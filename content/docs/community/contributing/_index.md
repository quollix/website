## Spread the Word

You can also support Quollix by helping more people learn about the project:

* talk with potentially interested friends, colleagues, bosses, or organizations
* share it on social media
* write blogs or reviews where they are useful
* suggest other practical ways to reach people who may benefit from Quollix

## Improve the Website

{{< alert title="Note" color="warning" >}}
GitHub is the right place for contributions and development discussions, but not for user support requests.
{{< /alert >}}

Website contributions are welcome as well. The website repository is [github.com/quollix/website](https://github.com/quollix/website). Contributing website content does not require coding: basic Markdown knowledge and knowing how to contribute through GitHub are sufficient. Read [Website Contributions]({{< relref "website.md" >}}) for website-specific contribution information.

## Expand the App Store

{{< planned-feature >}}

If you have software with a web interface that you would like to share with the community, you will be able to package it as an app and upload it once public publishing is available. If an app becomes popular, we will consider adopting it as an official app.

## Improve App Compatibility

You can also contribute by improving software available in the App Store, so that it integrates better with Quollix. See the [App Design Recommendations]({{< relref "docs/app-store/app-design-recommendations.md" >}}) for the expected integration model. Common high-value improvements are:

* OIDC support
* OIDC client configuration through environment variables

## Improve Security

Security-related contributions are useful as well. Security checks, responsible reporting of potential security issues, and proposals for improving the security model are welcome. Please follow the [Security Policy]({{< relref "docs/legal/security.md" >}}) when reporting vulnerabilities.

## Support Development

Buying a [license key]({{< relref "docs/pricing/purchase-premium.md" >}}) helps fund Quollix development and the infrastructure used by the project.
